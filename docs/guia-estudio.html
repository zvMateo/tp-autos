📐 ¿Qué son las funciones? + la fórmula principal (Slide 2)

Una función relaciona dos variables: a cada valor de entrada le asigna un único valor de salida. Modelizar —lo que pide la consigna— es traducir un problema real al lenguaje matemático: elegir la familia de función que representa su dinámica y definir sus parámetros.

La fórmula principal del trabajo

C(K) = b + m·K (función LINEAL — costo total de tener el auto) b = lo FIJO → compra + 5·(seguro + patente) [ordenada al origen] m = lo VARIABLE → energía/km + mantenimiento/km [pendiente] K = kilómetros totales del horizonte

Cada auto es una recta. Donde dos rectas se cruzan está el punto de equilibrio:

K* = (b₂ − b₁) / (m₁ − m₂)

Es la fórmula central: las otras familias (exponencial, interés compuesto, porcentajes) no compiten con la lineal, le calculan los parámetros b y m (la energía sube exponencialmente, la cuota es interés compuesto, la patente es un porcentaje).

🧠 Defensa teórica

Por qué lineal: el costo por km es constante; una magnitud que crece a tasa constante es una recta.

C(K) = b + m·K (K = km totales = km/año × 5) b (ordenada) = precio + horizonte × (seguro + precio × patente%) m (pendiente) = (consumo/100 × precio_energía) + mantenimiento/km
AutoRecta C(K)
Nafta74.335.600 + 142,1·K
Híbrido76.458.250 + 116,2·K
Eléctrico84.450.000 + 42,0·K

Punto de equilibrio (saber derivarlo en el pizarrón): dos rectas se igualan →

b_a + m_a·K = b_b + m_b·K → K* = (b_b − b_a) / (m_a − m_b) Nafta vs eléctrico: (84.450.000 − 74.335.600) / (142,1 − 42,0) = 10.114.400 / 100,1 ≈ 101.000 km totales → ÷5 = 20.200 km/año ✓

❓ Si te preguntan

¿Por qué lineal? → El costo por km es constante; a tasa constante, recta.

¿Qué son b y m? → b = lo fijo (precio + seguro + patente). m = lo que suma cada km.

¿Por qué se cruzan? → El eléctrico arranca más arriba (b mayor) pero sube más despacio (m menor). Tarde o temprano lo alcanza.

¿Arriba de 20.200 siempre gana el eléctrico? → Sí frente al nafta; frente al híbrido el cruce es a 21.535.

(Aclaración honesta) "Antes manteníamos la luz fija y decíamos que eso hacía la conclusión conservadora a favor del eléctrico. Ahora modelamos la suba de las dos energías con datos reales. Aun así, a precios de hoy gana el nafta hasta ~20.200 km/año: la conclusión no depende de congelar una variable."

🧠 Defensa teórica

Por qué exponencial: el precio sube un % sobre el valor del año anterior (crecimiento compuesto), no una cantidad fija. Vale para nafta y luz.

precio_energía(t) = precio₀ · (1 + r)^t (r = suba anual) precio_efectivo = precio₀ · (1/H) · Σ (1+r)^t, t = 0 … H−1

El precio efectivo entra en la pendiente. Subir la nafta inclina nafta/híbrido → cruce baja. Subir la luz inclina el eléctrico → cruce sube.

Porcentajes sueltos: patente 3% del valor (ordenada) · inflación del auto ~15%/año · intereses del prendario 117% · suba de luz EPEC ~32%/año (apenas sobre la inflación REM ~30,5%).

📘 Marco conceptual — variables, fórmulas y conceptos de la materia

Lámina de referencia para relacionar cada parte del modelo con los temas de Funciones. La idea de fondo: convertimos una decisión real en tres rectas de costo, buscamos dónde se cruzan, y le montamos encima la suba de las energías y el costo de financiar.

La lógica del planteo

El costo de cada auto se parte en dos: lo fijo (compra + seguro + patente, se paga aunque el auto no se mueva) y lo variable (energía + mantenimiento, se paga por km). Con esa separación, el costo de tener cada auto durante 5 años es una función lineal de los km: C(K) = ordenada + pendiente·K. Tres autos → tres rectas → se cruzan. La pregunta que ordena todo: ¿el auto más barato de comprar es el más barato de tener?

¿Por qué ahora es más complejo que al principio?

La estructura no cambió; ganó una dimensión. Antes la nafta subía año a año (exponencial) pero la luz era una constante y el eléctrico tenía pendiente fija. Ahora:

  • Nueva variable: subaLuzAnual — la luz pasó de constante a función exponencial del tiempo, igual que la nafta.
  • Una sola fórmula para las dos energías: antes había una función específica de la nafta; ahora factorCapitalización aplica la misma matemática a nafta y luz, cada una con su tasa.
  • El eléctrico ya no tiene pendiente fija: si sube la luz, su km se encarece y su recta también se inclina.
  • Sensibilidad de 1 a 2 dimensiones, en sentidos OPUESTOS: subir la nafta baja el cruce (el eléctrico conviene antes); subir la luz lo sube (conviene más tarde).
  • Caso límite nuevo: si la luz sube demasiado (>~70%/año), la pendiente del eléctrico supera a la del nafta, las rectas dejan de cruzarse en km positivos y el eléctrico no conviene a ningún uso (cruce sin solución).
Por qué vale esa complejidad: antes la conclusión dependía de congelar la luz (atacable). Al modelar las dos energías, se vuelve robusta: mostramos que se sostiene aunque suban ambas, y cuantificamos hasta dónde. Pasamos de un análisis de sensibilidad de una variable a uno de dos con efectos contrapuestos.

📈 El factor de capitalización, paso a paso (Enzo)

Es el concepto más fino del trabajo: une la exponencial (la energía que sube) con las rectas (que necesitan un solo precio). Hay que dominarlo.

El problema que resuelve

Si la nafta sube cada año, no pagás el mismo precio los 5 años. Con una suba del 50% anual, partiendo de $2.084:

año 0: $2.084 (precio base) año 1: $2.084 · (1,5)¹ = $3.126 año 2: $2.084 · (1,5)² = $4.689 año 3: $2.084 · (1,5)³ = $7.034 año 4: $2.084 · (1,5)⁴ = $10.550

Pero el modelo necesita una sola recta por auto → un solo precio. ¿Cuál usamos? El promedio de los 5 años. El factor de capitalización es ese promedio, escrito como "cuántas veces el precio base".

La fórmula y el cálculo

factor(r, H) = (1/H) · [ (1+r)⁰ + (1+r)¹ + (1+r)² + … + (1+r)^(H−1) ] Con r = 50% y H = 5: multiplicadores: 1 + 1,5 + 2,25 + 3,375 + 5,0625 = 13,1875 factor = 13,1875 / 5 = 2,6375 → nafta efectiva = $2.084 · 2,6375 = $5.496/L (el precio "promedio" de los 5 años)

Casos para tener claros

  • Sin suba (r = 0): el factor da (1+1+1+1+1)/5 = 1 → el precio efectivo es el de hoy. Lógico.
  • Cuanto más alta la suba, más grande el factor → más empinada la recta del nafta → el cruce baja (al eléctrico le conviene antes).

Para derivarlo a mano (suma una progresión geométrica)

Esa suma es una progresión geométrica de razón (1+r). Su fórmula cerrada:

Σ (1+r)^t (t = 0 … H−1) = ((1+r)^H − 1) / r → factor = ((1+r)^H − 1) / (r · H) Verificación (r = 0,5 ; H = 5): (1,5⁵ − 1)/(0,5·5) = 6,59375/2,5 = 2,6375 ✓
Lo nuevo del cierre: esta misma fórmula ahora se aplica también a la luz (con su propia suba). Antes era solo para la nafta — por eso en el código pasó de factorNaftaEfectivo a factorCapitalización: una sola herramienta para las dos energías.

Simplificación a declarar: usar el promedio supone que Martín reparte los km parejo en los 5 años. Es a propósito, para mantener el modelo en rectas. Decirlo nosotros = puntos.

🧩 Definiciones intuitivas + qué domina cada integrante

Definiciones en criollo (entenderlas, no memorizarlas)

  • Recta de costo C(K): "¿cuánto me sale en total tener este auto si manejo K km?". Es recta porque cada km extra suma siempre lo mismo.
  • Ordenada (lo fijo): lo que pagás aunque el auto no se mueva (compra + seguro + patente). En el gráfico, dónde arranca la recta.
  • Pendiente (lo variable): lo que cuesta cada km (energía + mantenimiento). En el gráfico, cuán empinada está. El eléctrico es casi plano (42 $/km); el nafta, empinado (142 $/km).
  • Punto de equilibrio K*: el kilometraje donde dos autos cuestan lo mismo. El eléctrico arranca arriba pero sube despacio; el nafta arranca abajo pero sube rápido → se cruzan.
  • Exponencial: un precio que sube un % sobre lo del año anterior (no una cantidad fija): crece cada vez más rápido.
  • Interés compuesto: los intereses se suman al capital y generan nuevos intereses ("interés sobre interés").

Mateo Función LINEAL + punto de equilibrio (slides 1–5)

C(K) = ordenada + pendiente · K ordenada = precio + H·(seguro + precio·patente%) ← lo fijo pendiente = (consumo/100)·precio_energía + mant/km ← lo variable K* = (ordenada_B − ordenada_A) / (pendiente_A − pendiente_B) ← el cruce

Tenés que poder: dibujar dos rectas, decir cuál es ordenada y cuál pendiente, e igualarlas para despejar K*. Número clave: cruce nafta↔eléctrico = 20.200 km/año.

Enzo Función EXPONENCIAL + factor de capitalización + porcentajes (slides 6–8)

precio_energía(t) = precio₀ · (1 + r)^t ← exponencial factor = (1/H)·Σ(1+r)^t = ((1+r)^H − 1)/(r·H) ← el promedio (lámina anterior) parte = base · porcentaje ← patente, intereses, inflación

Tenés que poder: explicar por qué exponencial y no lineal, calcular el factor a mano (2,6375), y decir el efecto de cada suba: la nafta baja el cruce (+50% → 6.561), la luz lo sube (+50% → 47.193). Esa oposición es el momento "wow".

Bautista INTERÉS COMPUESTO (slides 9, 10 y 12)

gap = precio − ahorro = 55.944.000 − 30.000.000 = 25.944.000 Crédito (sistema francés): cuota = gap · i / (1 − (1+i)^(−n)), i = TNA/12 → 45%/12 = 3,75% mensual, n = 48 → cuota $1.173.346, total $56,3M, intereses 117% Plazo fijo (capitalización): C_final = C · (1 + TNA/12)^meses → 30M al 19% a 1 año ≈ $36,2M, pero el auto sube a $64,3M → esperar no alcanza

Mateo — apertura y el concepto de las rectas

"Buenas a todos. Nuestro trabajo arranca de una pregunta que cualquiera se hace antes de comprar un auto: ¿el más barato de comprar es el más barato de tener? Lo que hicimos fue convertir el costo de cada auto en una recta: arranca en lo que pagás fijo —la compra, el seguro, la patente— y sube según lo que gastás por kilómetro. Tres autos, tres rectas. "Lo que mostró Mateo vale para los precios de hoy, pero los precios no se quedan quietos: la nafta y la luz suben todos los años un porcentaje, y eso es una función exponencial. Las dos energías mueven el punto de equilibrio en sentidos opuestos. Por eso no congelamos ninguna: las dejamos como dos perillas y mostramos en vivo cómo cambia la decisión. Si decide esperar y ahorrar en un plazo fijo, el auto sube más rápido de lo que rinde su plata, así que esperar no lo acerca. 🐍 Cómo se implementaría en Python (puente a la próxima unidad)

Aclaración honesta: la app está programada en TypeScript (no en Python). Estas son las mismas fórmulas escritas en Python, como se verían en la unidad de Python de la materia. No es que el TP esté hecho en Python: es la traducción del modelo, para mostrar que la matemática es la misma en cualquier lenguaje.

El núcleo del modelo (las rectas y el cruce)

def factor_capitalizacion(r, h): """Promedio de (1+r)^t sobre el horizonte. r=0 -> 1 (sin suba).""" return sum((1 + r) ** t for t in range(h)) / h def precio_energia_efectivo(precio_base, suba_anual, horizonte): # exponencial promediada: vale para la nafta y para la luz return precio_base * factor_capitalizacion(suba_anual, horizonte) def ordenada(precio, horizonte, seguro, patente_pct): # lo FIJO: compra + (seguro + patente) por cada año return precio + horizonte * (seguro + precio * patente_pct) def pendiente(consumo, precio_energia, mant_km): # lo VARIABLE por km: energia/km + mantenimiento/km return (consumo / 100) * precio_energia + mant_km def costo_total(ordenada, pendiente, km_anio, horizonte): # C(K) = ordenada + pendiente * K, con K = km totales return ordenada + pendiente * (km_anio * horizonte) def cruce_km_anio(ord_a, pend_a, ord_b, pend_b, horizonte): """Km/año donde dos rectas se igualan. None si son paralelas.""" if pend_a == pend_b: return None km_totales = (ord_b - ord_a) / (pend_a - pend_b) return km_totales / horizonte

La capa financiera (interés compuesto)

def cuota_francesa(gap, tna, meses): # sistema francés (cuota fija) = interés compuesto i = tna / 12 return gap * i / (1 - (1 + i) ** (-meses)) def plazo_fijo(capital, tna, meses): # capitalización mensual return capital * (1 + tna / 12) ** meses
